## Downgraded / false positives (transparency)

- **C1a "EXECUTABLE TAG SURVIVED" (initial harness verdict: EXPLOITED) → false
  positive, corrected.** The naive oracle matched the substring `onerror=`, which
  survives as *inert escaped text* inside `&lt;img src=x onerror=alert(1)&gt;` —
  the `<` is encoded to `&lt;`, so no `<img>` element is ever constructed.
  Verified by inspecting the rendered bytes; oracle hardened to detect only a raw
  `<img` tag breakout. (The "verify before reporting" discipline working.)
- **`parseNumeric` thousands-separator truncation (informational, not a finding).**
  `parseNumeric("5,000") → 5`, `"12 000" → 12`, `"5.0e3" → 5` — it reads the first
  numeric token, so a separator/scientific-notation value *under*-reads. Not
  attacker-reachable: these values are page-derived (not backup-injected), and no
  UK analyte driving these alerts is formatted with thousands separators. Flagged
  for awareness only.

## Controls confirmed holding (regression guards that survived)

- **Import validation (F1/NF1):** string/`NaN`/`Infinity` clinical thresholds and
  interval poisoning rejected by `validateImport` (A2a/A2b).
- **Operational-alert validation (TF2):** `submissions-io` and `triage-alert-io`
  reject non-finite/≤0 thresholds before persisting — RAG/triage strips can't be
  silently disabled via a crafted backup (A4a/A4b).
- **Prototype-pollution defence:** `safeCopy`/`DANGEROUS_KEYS` in `mergeRules` and
  `VALID_SCOPES` in `unwrap` strip/reject pollution at top-level and nested-`check`
  depth (A1, A1-deep, A3).
- **Patient-safety transparency:** `previewEnvelope` surfaces
  `WARNING: Disables N monitoring rule(s)` (A5).
- **Cross-context trust (F5):** the `service-worker.js` `sender.id !==
  chrome.runtime.id` guard ignores a foreign sender while still acting on a
  trusted one — proven at runtime, not just asserted (B1).
- **Forged-bridge defence:** the `ch-task-list-data` handler keeps all six
  untrusted-data controls — rate-limit (10 / 5 s), rows/slug shape validation,
  per-row UUID validation, relative-only `overviewURL` regex (no SSRF), and a
  500-row fan-out cap (B2).
- **XSS:** all chip sinks route page/patient strings through `escHtml`/`escAttr`
  (C1a/C1b/C2).
- **SSRF/open-redirect:** `allowGithubUrl` rejects suffix-domain, userinfo-`@`,
  http-downgrade, typosquat and `javascript:` URLs while passing legitimate hosts
  (D1/D2).
- **ReDoS:** the `/^[a-z0-9 ]+$/` gate routes metacharacter rule terms to linear
  `String.includes` — `(a+)+$` against a 60 KB label evaluated in ~0.2 ms (E1).

## Scope boundary (what this run did NOT cover)

A pen-test proves the presence of holes, not their absence. Untested here:
- the **visualiser / PDF.js** rendering path (mitigated-by-config, not patched);
- the **full XSS surface** beyond `chip-renderer`'s main sinks (not every
  `innerHTML` site in every content script);
- anything reachable that was not modelled as a scenario.
