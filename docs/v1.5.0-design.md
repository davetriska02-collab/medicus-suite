# Medicus Suite v1.5.0 — Feature Design

## Overview

Three independent features implemented in dependency order.

---

## Feature 1 — Slot-Type Alert Ribbon

### Problem
The Slots tab shows counts per appointment type but there is no way to be alerted when
a critical type (e.g. "Duty Doctor", "Same Day Urgent") runs out or drops dangerously low.
Staff must remember to glance at the panel; the data does not push itself into attention.

### Design
A coloured alert ribbon is rendered at the top of the Slots module content area whenever one
or more user-defined alert rules are triggered. Rules are evaluated entirely client-side after
each API fetch — no extra network calls.

**Storage key:** `slots.alertRules`  
**Schema:** `Array<{ id: string, typeName: string, threshold: number, enabled: boolean }>`

**Trigger logic:**
- `count <= threshold && threshold === 0` → level `red`   ("zero slots")
- `count <= threshold && threshold > 0`  → level `amber`  ("low slots")
- Not triggered when `count > threshold`

**Ribbon rendering:**
```
┌────────────────────────────────────────────────────────┐
│ ⚠  Duty Doctor: 1 slot remaining  ·  Urgent: 0 slots  │
└────────────────────────────────────────────────────────┘
```
Ribbon colour: red if any rule is at level `red`, otherwise amber.  
Multiple triggered rules are listed inline, separated by `·`.

**Options page (Slot Counter section):**
A simple rule builder — one row per rule with: slot-type text input, threshold number input,
enabled toggle, delete button. "Add rule" button appends a new blank row.
Saved to storage on every change (no explicit save button needed for individual toggles;
explicit Save button for type/threshold edits).

### Files
- Modify: `side-panel/modules/slots/slots.js` (+~60 lines)
- Modify: `side-panel/modules/slots/slots.css` (+~25 lines)
- Modify: `shared/io/slot-counter-io.js` (+~15 lines)
- Modify: `options/options.html` (+~40 lines)
- Modify: `options/options.js` (+~100 lines)

## Feature 2 — Pop-out / Free-Float Window

### Problem
The Chrome side panel is docked to the right edge of a single browser window and disappears
when the browser is in fullscreen or when the user switches to another window. For practices
with dual monitors or who use the panel heavily, a floating window that can be moved to a
second screen is much more useful.

### Design
A "Pop out" button (⊞) is added to the side panel nav. Clicking it opens `pop-out/pop-out.html`
in a `chrome.windows.create({ type: 'popup' })` window. The pop-out page reuses all the same
module JS/CSS; it is functionally identical to the side panel.

Only one pop-out window is permitted at a time. If the window is already open, the button
becomes a "Focus" button that calls `chrome.windows.update(id, { focused: true })`.

**Window persistence:**
- Last position and size stored in `popout.windowState` (`{ left, top, width, height }`)
- Restored on next open via `chrome.windows.create({ left, top, ... })`
- Updated on `chrome.windows.onBoundsChanged` (service worker)

**`shared/popout-manager.js`:**
- `open()` — create or focus
- `close()` — close if open
- `getState()` — read `popout.windowId` from storage
- Exposed as `window.PopoutManager`

**`shared/io/popout-io.js`:**
- `popoutExport()` / `popoutImport()` — include window state in backup/restore

**Service worker changes:**
- Listen for `chrome.windows.onRemoved` to clear `popout.windowId` when the user closes the window
- Handle `popout:open` message from panel

**`pop-out/pop-out.html`:**  
Minimal shell: loads `pop-out/pop-out.css` + all shared scripts + `pop-out/pop-out.js`.  
Omits the side-panel outer nav chrome (brand, scroll arrows, settings button) — replaces it
with a smaller title bar showing "Medicus Suite" and a close button.

**`pop-out/pop-out.js`:**  
Mirrors `panel.js` module registration and switching logic. No WR/RM strips (they remain in
the docked panel; duplicate polling is wasteful). Starts on `slots` module by default; stores
`popout.activeModule` separately so each panel can be on a different tab.

**Manifest changes:**
- Add `"windows"` to `permissions`
- Add `pop-out/` resources to `web_accessible_resources`

### Files
- New: `pop-out/pop-out.html`
- New: `pop-out/pop-out.js`
- New: `pop-out/pop-out.css`
- New: `shared/popout-manager.js`
- New: `shared/io/popout-io.js`
- Modify: `side-panel/panel.html` — popout button
- Modify: `side-panel/panel.js` — wire popout button + respond to window close
- Modify: `service-worker.js` — `chrome.windows.onRemoved` listener + `popout:open` message
- Modify: `manifest.json` — permissions + web_accessible_resources

## Feature 3 — Triage Remaining-Capacity Alert

### Problem
The request monitor strip already shows task counts, but there is no alerting when those
counts exceed safe working levels. A triage team managing 50 new medical requests with no
visual escalation gets no signal that they are overwhelmed.

### Design
Two new files implement a stateless rules evaluator and storage IO. The panel's existing
request-monitor strip is enhanced to evaluate rules after each poll and highlight when any
threshold is breached. A Chrome desktop notification fires once per session per threshold
crossing (de-duplicated by bucket key + poll timestamp).

**`engine/triage-alert-engine.js`:**
```
evaluate(buckets, rules)
  → { triggered: [{ key, label, count, threshold, level }], maxLevel: null|'amber'|'red' }
```
Level logic:
- `count >= threshold * 2` → `'red'`
- `count >= threshold`     → `'amber'`

**`shared/io/triage-alert-io.js`:**  
Storage key: `suite.triageAlert.rules`  
Default rules (disabled by default):
```
medNew:     threshold 10, label "New medical"
medReply:   threshold 5,  label "Medical reply"
adminNew:   threshold 10, label "New admin"
adminReply: threshold 5,  label "Admin reply"
```

**Panel changes:**
After `renderRmStrip()`, call `applyTriageAlerts(result.buckets)`:
- Adds `.rm-strip-alerted-amber` or `.rm-strip-alerted-red` class to `#rmStrip`
- For `red` level: fires Chrome notification (once per bucket crossing, tracked via
  `lastAlertedBucket` map in panel-scope memory — clears on page reload)

**Options page (Suite section, below Request Monitor):**
"Triage capacity alerts" sub-section. Shows one row per bucket with:
threshold number input (disabled when rule is disabled), enabled toggle, and the current
live count (updated from storage state). Saving writes to `suite.triageAlert.rules`.

### Files
- New: `engine/triage-alert-engine.js`
- New: `shared/io/triage-alert-io.js`
- Modify: `side-panel/panel.js` (+~50 lines)
- Modify: `side-panel/panel.css` (+~15 lines)
- Modify: `options/options.html` (+~50 lines)
- Modify: `options/options.js` (+~80 lines)
