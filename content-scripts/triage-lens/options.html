About Triage Lens

Triage Lens is a personal-use overlay for Medicus. It surfaces clinical signals on patient records, classifies and decorates triage queue rows, and adds a request-context panel on task detail pages. All processing is local — no data leaves the browser.

How rules work

Each rule has a label, a severity colour, one or more text patterns, and zero or more actions. When a rule's pattern matches the configured fields on a configured page, the chip appears on the HUD or queue row. If the chip has actions, click it to expand a menu of click-to-fire shortcuts.

  • Link — opens a URL in a new tab (NICE guidelines, questionnaires, local SOPs).
  • Snippet — copies a text template to the clipboard (history-taking, examination, safety-net wording).
  • Note — shows informational text in a popover (monitoring reminders, red-flag checklists).

Safety

This is display-only. It surfaces existing record entries with provenance — no synthesis, no inference of unrecorded clinical states, no recommendations. Actions never auto-fire — every action is click-to-confirm.

Authoring tips

  • If your patterns are simple words or phrases, leave regex off — Triage Lens will escape special characters and match whole-word.
  • For phrases with optional parts (e.g. self.?harm matching "self-harm" / "self harm"), enable regex.
  • Queue page only has request text available. Other fields are only on detail / record pages.
  • Use the live preview tab to test patterns before saving.
