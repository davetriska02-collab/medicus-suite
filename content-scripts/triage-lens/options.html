Analyte-threshold rules escalate investigation result severity — they can raise a chip from none to amber or red, but never lower a lab-flagged result. Imported rules arrive disabled; review each rule's analyte match strings and threshold values before enabling.

Scope. These built-in rules are a safety-net layer covering common acute biochemistry, FBC indices and therapeutic drug levels — electrolytes, eGFR, platelets, neutrophils, INR, calcium, magnesium, lithium, digoxin, TSH and HbA1c — plus key microbiology text rules (MSU, blood culture, bowel screening). They are not a complete results-interpretation system: an analyte with no rule (for example ferritin, B12/folate or LFTs) simply shows the lab's own flag, so an absent chip is not a clinical all-clear. Add your own rule for anything else your practice wants flagged.

How result rules work:
Rules escalate severity only — they can raise a result chip to amber or red, but never lower a lab-flagged result. Thresholds are evaluated inclusively: "at or above (≥)" fires when value ≥ threshold; "at or below (≤)" fires when value ≤ threshold. For "at or above", set red ≥ amber (red fires at a higher value). For "at or below", set red ≤ amber (red fires at a lower value). At least one threshold must be set. Analyte matching is case-insensitive substring against the result name as reported by your lab. Analyte exclude (optional) skips look-alike analytes that share a match token — e.g. a "platelet" rule with exclude "mean platelet" won't fire on "Mean platelet volume", and a serum potassium rule with exclude "urine" won't fire on a urine potassium.

Text / culture rules match a result by analyte name, then inspect the result text: if any normal phrase is found (case-insensitive) the calm "normal" chip is shown; otherwise the label chip is shown (amber by default). Use this for microbiology culture reports such as MSU.

Specimen scope (optional) restricts a rule to results that belong to a named specimen group — for example scoping a "Culture" rule to "THROAT SWAB" so it never fires on urine or blood culture lines. Leave blank to match any specimen. Fail-open: if the lab report carries no group header the scope is ignored.

Combo rules fire only when every condition matches within the same report (logical AND) — e.g. sterile pyuria: pus cells above 40 AND culture shows no growth. Each condition carries its own analyte match / specimen. Combo is escalate-only, defaults to amber (review), and should stay disabled until a clinician has reviewed it.

No rule matched? The result still shows the lab's own flag — but an absent rule chip is not a clinical all-clear: it only means no rule covered that analyte.

About Triage Lens

Triage Lens is a personal-use overlay for Medicus. It surfaces clinical signals on patient records, classifies and decorates triage queue rows, and adds a request-context panel on task detail pages. All processing is local — no data leaves the browser.

How rules work

Each rule has a label, a severity colour, one or more text patterns, and zero or more actions. When a rule's pattern matches the configured fields on a configured page, the chip appears on the HUD or queue row. If the chip has actions, click it to expand a menu of click-to-fire shortcuts.

  • Link — opens a URL in a new tab (NICE guidelines, questionnaires, local SOPs).
  • Snippet — copies a text template to the clipboard (history-taking, examination, safety-net wording).
  • Note — shows informational text in a popover (monitoring reminders, red-flag checklists).

Safety

This is display-only. It surfaces existing record entries with provenance — no synthesis, no inference of unrecorded clinical states, no recommendations. Actions never auto-fire — every action is click-to-confirm.

Authoring tips

  • If your patterns are simple words or phrases, leave regex off — Triage Lens will escape special characters and match whole-word.
  • For phrases with optional parts (e.g. self.?harm matching "self-harm" / "self harm"), enable regex.
  • Queue page only has request text available. Other fields are only on detail / record pages.
  • Use the live preview tab to test patterns before saving.
